Resumen visual de Git — conceptos y flujo aprendido

Git es un sistema que guarda el historial de cambios de tu código. Como un historial de guardados de videojuego — puedes volver a cualquier punto.

Los tres conceptos clave

Repositorio

Tu carpeta de proyecto con Git activado. Tiene una carpeta oculta .git donde se guarda todo el historial.

Commit

Una foto del estado de tu proyecto en un momento concreto. Cada commit tiene un mensaje y un identificador único.

Rama (master)

La línea principal de desarrollo. Se crea sola al hacer git init. Por ahora solo trabajas aquí.

Flujo del día a día

local
git status
Ver qué archivos han cambiado desde el último commit. Siempre el primer paso.
local
git add .
Prepara los cambios para el commit. El punto significa "todo". Solo añade lo que ha cambiado, no el proyecto entero.
local
git commit -m "mensaje"
Hace la foto. Guarda los cambios en el historial local. El mensaje describe qué hiciste.
remoto
git push
Sube los commits locales a GitHub. El primer push fue con -u origin master para configurar el destino — ya no hace falta.

El resto de comandos

Solo una vez

git config Guarda tu nombre y email. Se hace al instalar Git.
git init Activa Git en la carpeta del proyecto.
git remote add Conecta la carpeta local con GitHub.

Consulta y sincronía

git log Ver el historial de commits con fecha y mensaje.
git pull Bajar cambios de GitHub a tu máquina.
Autenticación: Git Credential Manager (incluido en Git para Windows) guardó tu token de GitHub la primera vez. Por eso no te pide contraseña en cada push.
